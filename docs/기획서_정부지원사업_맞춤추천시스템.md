# 정부지원사업 맞춤추천 시스템 기획서

> **문서 버전**: v1.0  
> **작성일**: 2026-04-23  
> **프로젝트명**: 정부지원사업 맞춤형 추천 시스템

---

## 1. 프로젝트 개요

### 1.1 목적

기업 정보를 입력받아 K-Startup, 기업마당에서 수집한 정부지원사업 공고 중 해당 기업에 적합한 공고를 자동으로 추천하고, 맞춤형 보고서를 생성하는 웹 서비스.

### 1.2 핵심 사용자

정부지원사업을 찾는 **기업 대표자**

### 1.3 핵심 가치

> "내가 신청할 수 있는 지원사업이 뭐야?"를 빠르게 답해주는 것

### 1.4 기술 스택

| 구분 | 기술 |
|------|------|
| 프론트엔드 | Next.js 16 (App Router), React 19, Tailwind CSS 4, TypeScript |
| 백엔드 | Python 3.9+ (subprocess 호출 방식) |
| OCR | Tesseract.js (클라이언트), Claude Vision API (선택) |
| 크롤링 | requests, BeautifulSoup4, lxml |
| 보고서 | pandas, openpyxl (Excel), reportlab (PDF) |
| 연동 방식 | Server Action → Python subprocess |

## 3. 사용자 플로우

### 3.1 전체 흐름

```
[Step 1] 사업자등록증 업로드 → OCR → 폼 자동입력
   또는 직접입력 선택
        ↓
[Step 2] 나머지 정보 수동 입력/수정
   - 대표자 생년월일, 성별
   - 사업아이템 한줄정리
        ↓
[Step 3] 추천 결과 웹에서 조회
   - 카드 리스트, D-day 뱃지
   - 필터/정렬 (적합도순, 마감임박순, 분야별, 지역별)
   - 상세 모달 (매칭 사유, 공고 원문)
        ↓
[Step 4] 보고서 다운로드 (Excel/PDF)
```

### 3.2 Step 1: 데이터 입력

#### OCR 업로드

- **지원 형식**: PNG, JPEG, WebP, PDF
- **처리 방식**: Tesseract.js (클라이언트 측, API 키 불필요)
- **UI**: 드래그앤드롭 영역 + 파일 선택 버튼
- **상태 전이**: 기본 → 드래그 중 → 처리 중 → 완료

#### OCR 추출 필드

| 추출 필드 | 추출 방식 | 예시 |
|----------|----------|------|
| 사업자등록번호 | `(\d{3})-(\d{2})-(\d{5})` 패턴 | 123-45-67890 |
| 기업명 | "상호:", "법인명:" 뒤 텍스트 | CV기획 |
| 대표자명 | "대표자:", "성명:" 뒤 텍스트 | 홍길동 |
| 소재지 | "사업장 소재지:", "주소:" 뒤 텍스트 | 서울시 강남구... |
| 설립연월일 | "개업연월일:" → YYYY-MM-DD 정규화 | 2020-01-01 |
| 주업종(업태) | 알려진 업태 키워드 매칭 | 정보통신업 |
| 주요산업(종목) | "~업" 패턴 + 알려진 종목 키워드 | 소프트웨어 |

#### 직접입력

OCR 없이 폼에서 수동으로 모든 필드 입력 가능.

### 3.3 Step 2: 정보 수정 및 보완 입력

| 섹션 | 필드 | 필수 | 비고 |
|------|------|------|------|
| 기업 기본정보 | 기업명 | O | OCR 자동입력 |
| | 설립연월일 | | OCR 자동입력 |
| | 소재지 | | OCR 자동입력 → 시/도 자동 추출 |
| | 주업종 | | OCR 자동입력 |
| | 주요산업 | | OCR 자동입력 |
| | 사업아이템 한줄정리 | | textarea, 매칭 관련도에 핵심 |
| 대표자 정보 | 대표자명 | | OCR 자동입력 |
| | 생년월일 | | 수동 입력 (연령 필터용) |
| | 성별 | | 라디오 버튼 (M/F) |

### 3.4 Step 3: 추천 결과

#### 결과 화면 구성

- **상단**: 기업명 + 추천 통계 (전체 공고 수, 추천 공고 수)
- **정렬**: 적합도순 / 마감임박순
- **필터**: 지원분야, 지역, 마감예정, 주관처
- **뷰 모드**: 카드뷰 / 리스트뷰 토글
- **데이터 최신성**: 캐시 나이 표시 + "최신 데이터로 추천받기" 버튼

#### 공고 카드

| 요소 | 설명 |
|------|------|
| 적합도 시그널 바 | 1~5 레벨 막대 (색상 구분) |
| D-day 뱃지 | 빨강(D-3 이내) / 주황(D-7 이내) / 회색 |
| 분야 뱃지 | 분야별 색상 매핑 |
| 공고 제목 | 클릭 시 상세 모달 |
| 주관기관 | |
| 접수기간 | |

#### 상세 모달

- 적합도 점수 + 시그널 바
- 매칭 사유 목록
- 접수 정보 (기간, 연락처)
- 신청 자격 (연령, 업력, 지역, 대상)
- 공고 내용 (contentText)
- 첨부파일 다운로드 링크
- 원본 공고 바로가기

### 3.5 Step 4: 보고서 다운로드

#### Excel 보고서 (4시트 구성)

| 시트 | 내용 |
|------|------|
| 기업 프로필 | 기업명, 대표자, 연령, 업력 등 기업 정보 요약 |
| 추천 공고 | 순위, 적합도 바, 사업명, 분야, 주관기관, 접수기간, 추천사유 |
| 전체 공고 | 필터링 없이 수집된 모든 공고 목록 |
| 통계 | 분야별, 지역별, 기관구분별 건수 |

#### PDF 보고서

- 제목 (기업명 + 생성일)
- 기업 프로필 테이블
- 추천 공고 카드형 레이아웃
- 한글 폰트: macOS(AppleSDGothicNeo), Linux(NanumGothic)

### 4.3 크롤링 설정

| 항목 | 값 | 설명 |
|------|---|------|
| MAX_PAGES | 10 | 최대 크롤링 페이지 수 |
| REQUEST_DELAY | 2초 | 목록 페이지 간 대기 |
| DETAIL_DELAY | 1초 | 상세 페이지 간 대기 |
| TIMEOUT | 30초 | 요청 타임아웃 |

### 4.4 캐시 전략

- **저장 경로**: `data/cache/{source}_{timestamp}.json`
- **캐시 구조**: `{ crawledAt, totalCount, source, items[] }`
- **로드 순서**: 최신 캐시 → 시드 데이터(result.json) 순
- **갱신**: 사용자가 "최신 데이터로 추천받기" 클릭 시 신규 크롤링

## 5. 매칭 엔진

### 5.1 매칭 프로세스 (2단계)

```
입력: 기업 프로필 + 공고 리스트
        ↓
  ┌─────────────────────────┐
  │  1단계: 자격 필터 (Hard)  │  → 부적합 공고 제외
  │  접수마감 / 연령 / 업력 / 지역 │
  └─────────┬───────────────┘
            ↓ (통과한 공고만)
  ┌─────────────────────────┐
  │  2단계: 적합도 평가 (Soft) │  → 0~100점 산정
  │  관련도 / 부합도 / 긴급도 / 신뢰도 │
  └─────────┬───────────────┘
            ↓
  ┌─────────────────────────┐
  │  시그널 레벨 변환 (1~5)   │
  └─────────┬───────────────┘
            ↓
  결과 정렬 (적합도 내림차순) → MatchResult[]
```

### 5.2 1단계: 자격 필터 (Hard Rules)

자격 필터는 **하나라도 실패하면 해당 공고를 결과에서 제외**한다. 점수 산정 없이 Pass/Fail 판정.

| # | 필터 항목 | 판정 기준 | 미충족 시 |
|---|----------|----------|----------|
| 1 | 접수 마감 | 접수 종료일 < 오늘 날짜 | 제외 |
| 2 | 대표자 연령 | 파싱된 연령 범위(min~max) 내 CEO 나이 포함 여부 | 제외 |
| 3 | 창업 업력 | 기업 업력이 공고 조건(N년 이내/이상) 충족 여부 | 제외 |
| 4 | 지역 | 공고 지역과 기업 소재지 일치 (전국은 무조건 통과) | 제외 |

**예외 처리 (건너뜀, 제외하지 않음)**:
- CEO 생년월일 미입력 → 연령 필터 건너뜀
- 공고의 대상연령이 "전체" → 건너뜀
- 공고나 기업의 지역 정보 없음 → 건너뜀

### 5.3 2단계: 적합도 평가 (Soft Scoring)

자격 필터를 통과한 공고에 대해 4개 차원의 점수를 산정하고 가중합으로 총점을 계산한다.

#### 가중치 배분

| 차원 | 가중치 | 비중 | 설계 의도 |
|------|--------|------|----------|
| 사업 관련도 | **45점** | 45% | 기업의 사업 내용과 공고의 관련성이 추천의 핵심 |
| 대상 부합도 | **25점** | 25% | 기업유형/성별 등 지원 대상 조건 반영 |
| 긴급도 | **15점** | 15% | 마감 임박 공고 우선 노출 |
| 정보 신뢰도 | **15점** | 15% | 정보가 충분한 공고 우대 |
| **합계** | **100점** | 100% | |

#### (1) 사업 관련도 (45점)

기업의 사업 내용과 공고의 지원 분야가 얼마나 일치하는지 토큰 기반으로 평가한다.

**입력 텍스트**:
- 기업 측: 사업아이템 한줄정리 + 주요산업 + 주업종
- 공고 측: 제목 + 지원분야 + 내용(상위 500자)

**산정 방식**:

```
1. 양쪽 텍스트를 토큰화 (2글자 이상 한글/영문, 불용어 제거)
2. 겹치는 토큰 수 계산
3. 일치율 = 겹치는 토큰 수 / 기업 토큰 수
4. 점수 = min(일치율 × 2.0, 1.0)
```

**점수 분포 (선형 비례, 50%에서 만점)**:

| 토큰 일치율 | 원점수 (0~1) | 배점 (×45) |
|------------|-------------|-----------|
| 0% | 0.0 | 0점 |
| 10% | 0.2 | 9점 |
| 25% | 0.5 | 22.5점 |
| 40% | 0.8 | 36점 |
| 50% 이상 | 1.0 (상한) | 45점 (만점) |

**불용어 목록** (21개):
> 사업, 공고, 모집, 지원, 년, 기업, 창업, 프로그램, 선정, 관련, 대상, 대한, 통한, 참여, 안내, 접수, 계획 등

#### (2) 대상 부합도 (25점)

기업유형과 CEO 성별이 공고의 지원 대상 조건과 얼마나 맞는지 평가한다.

**A. 기업유형 매칭 (최대 0.7점)**

기업 업력에 따라 유형을 분류하고, 공고의 target 필드와 비교한다.

| 기업 업력 | 분류되는 유형 |
|----------|-------------|
| 없음 / 0년 이하 | 예비창업자 |
| 1~6년 | 일반기업, 일반인, 창업기업 |
| 7년 이상 | 일반기업, 일반인 |

| 조건 | 점수 |
|------|------|
| 공고 대상 = "전체" 또는 빈 값 | +0.5 |
| 기업유형이 공고 대상에 포함 | +0.7 |
| 기업유형이 공고 대상에 미포함/불확실 | +0.2 |

**B. 성별 매칭 (최대 0.3점)**

공고 제목 + 내용에서 여성 관련 키워드를 탐지한다.

- **여성 패턴**: "여성", "우먼", "맘", "경력단절"
- **한정 키워드**: "한정", "대상", "전용", "우대"

| 조건 | 점수 |
|------|------|
| 성별 제한 없는 공고 | +0.15 |
| 여성 전용 공고 + 여성 CEO | +0.3 |
| 여성 전용 공고 + 남성 CEO | +0.0 (감점 없음) |

**최종 점수**: min(기업유형 + 성별, 1.0) × 25

#### (3) 긴급도 (15점)

접수 마감일까지 남은 일수에 따라 마감 임박 공고에 가점을 부여한다.

| 남은 기간 | 원점수 (0~1) | 배점 (×15) | 표시 |
|----------|-------------|-----------|------|
| D-3 이내 | 1.0 | 15점 | 마감임박 |
| D-4 ~ D-7 | 0.8 | 12점 | 마감임박 |
| D-8 ~ D-14 | 0.5 | 7.5점 | D-N |
| D-15 ~ D-30 | 0.3 | 4.5점 | |
| D-31 이상 | 0.2 | 3점 | |
| 마감일 불명 | 0.3 | 4.5점 | 중립 처리 |

#### (4) 정보 신뢰도 (15점)

공고의 핵심 필드가 얼마나 채워져 있는지 평가한다. 정보가 부족한 공고는 매칭 정확도가 낮으므로 감점한다.

**평가 대상 필드** (8개):

| # | 필드 | 설명 |
|---|------|------|
| 1 | targetAge | 대상 연령 |
| 2 | bizExperience | 창업 업력 |
| 3 | region | 지역 |
| 4 | target | 지원 대상 |
| 5 | receptionPeriod | 접수 기간 |
| 6 | contentText | 내용 본문 |
| 7 | supportField | 지원 분야 |
| 8 | supervisionOrg | 주관 기관 |

**산정 방식**: `점수 = (채워진 필드 수 / 8) × 15`

| 필드 채움률 | 배점 | 비고 |
|-----------|------|------|
| 100% (8/8) | 15점 | |
| 75% (6/8) | 11.3점 | |
| 50% (4/8) | 7.5점 | |
| 37.5% 미만 | 5.6점 이하 | "공고 정보 부족 (확인 필요)" 경고 |

### 5.4 총점 계산

```
총점 = (관련도 원점수 × 45) + (부합도 원점수 × 25) + (긴급도 원점수 × 15) + (신뢰도 원점수 × 15)
```

- 각 원점수: 0.0 ~ 1.0
- 총점 상한: 100점
- 소수점 1자리 반올림

**산정 예시**:

| 시나리오 | 관련도 | 부합도 | 긴급도 | 신뢰도 | 총점 | 레벨 |
|---------|--------|--------|--------|--------|------|------|
| 우수 매칭 | 0.8×45=36 | 1.0×25=25 | 1.0×15=15 | 1.0×15=15 | **91.0** | 5 |
| 양호 매칭 | 0.5×45=22.5 | 0.7×25=17.5 | 0.5×15=7.5 | 0.75×15=11.3 | **58.8** | 3 |
| 미흡 매칭 | 0.2×45=9 | 0.2×25=5 | 0.2×15=3 | 0.5×15=7.5 | **24.5** | 2 |

### 5.5 시그널 레벨 변환

총점을 사용자 친화적인 5단계 신호로 변환한다.

| 점수 구간 | 레벨 | 시그널 바 | 의미 |
|----------|------|----------|------|
| 80 ~ 100 | 5 | ▐▐▐▐▐ | 강력 추천 |
| 60 ~ 79 | 4 | ▐▐▐▐· | 적극 추천 |
| 40 ~ 59 | 3 | ▐▐▐·· | 권장 |
| 20 ~ 39 | 2 | ▐▐··· | 참고용 |
| 0 ~ 19 | 1 | ▐···· | 정보 제공 |

### 5.6 매칭 규칙 요약표

| 구분 | 항목 | 작동 방식 | 미충족 시 |
|------|------|----------|----------|
| **HARD** | 접수 마감 | 종료일 < 오늘 | 공고 제외 |
| **HARD** | 대표자 연령 | min ≤ 나이 ≤ max | 공고 제외 |
| **HARD** | 창업 업력 | 조건 부합 여부 | 공고 제외 |
| **HARD** | 지역 | 정확히 일치 (전국 통과) | 공고 제외 |
| **SOFT** | 사업 관련도 | 토큰 중복률 (45점) | 점수 감소 |
| **SOFT** | 대상 부합도 | 유형/성별 일치 (25점) | 점수 감소 |
| **SOFT** | 긴급도 | D-day 기반 (15점) | 점수 감소 |
| **SOFT** | 정보 신뢰도 | 필드 완성도 (15점) | 점수 감소 |
